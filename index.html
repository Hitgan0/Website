Projects

University Management System

The University Management System (UMS) is a full stack application that makes managing college easier for students and faculty by combining the platforms for course enrollment, tuition payment, and assignment submission into a singluar site. There are 3 roles: Admin, Instructor, and Student. Admins are able to create/edit/delete courses, sections for those courses, and users. Instructors are able to create/edit/delete assignments, links to course materials, and grades. Students are able to enroll into and drop courses, submit and edit assignments, pay tuition, and view and download their transcript.

The project is a Java Maven project with Spring dependencies. Spring Boot is used to manage dependency injection and Spring Security (along with JSON Web Tokens) are used to secure the roles, ensuring no access to restricted pages for each role. Spring's JDBC template is used to communicate with the MySQL backend and JUnit and Mockito are used to fully test the backend. For the frontend, React is used to create a responsive front-end. The Stripe API is used for tuition payment, HTML2PDF is used for transcript download, and MUI is used for course filtering.

This project was completed in a group of 4 over the course of 2 weeks, with an additional technical presentation at the end. It really put all of us to the test, having each of us work over 10+ hours each day just to meet deadlines. During that time I learned so much about not only what goes into making a full-stack application, but what I am capable of. Many times we had to learn on the go, and really take the learning into our hands, having to find our own solutions. There were many times where we didn't think we'd make a deadline, but we persevered together and put out a product we are all extremely proud of!

CPU Scheduling System

The CPU Scheduling System was a project that I completed in my 3rd year of college. It simulated processes waiting in memory for a CPU resource, using that resource, then waiting in I/O. As a group of 3, we implemented 4 CPU scheduling algorithms (First Come First Serve, Round Robin, Shortest Remaining Time, and Shortest Job First) to manage process execution order. The project was completed in C. Though my group did not end up scoring great in the final submission, this project contributed a lot to my personal growth. It taught me the value of professionalism, and keeping my composure through a stressful time. It also taught me about how important time management is. Multiple nights were spent staying up trying to implement one last feature or iron out a bug. If only we had the foresight of planning our schedule and minimizing distractions, maybe we could've scored better. Regardless, I am proud of this project because it acted as a stepping stone for me to grow into a more organized, logical, and well-rounded individual.
